# Business Plan — AgentPass

> ✅ = 検証済み | 🔄 = 進行中 | 📋 = 設計段階 | 🔭 = 仮説

---

## One-Liner

**「AIエージェントが人間と同じように経済活動する時代のインフラを先占する」**

---

## Problem

| 現在の課題 | 規模 |
|---|---|
| AIエージェントの爆発的普及 | GPT-4o, Claude 3.5, Gemini 等の自律化が加速 |
| M2M 決済インフラが存在しない | エージェントは「人間のAPIキー」を借りて動く |
| APIキー認証の脆弱性 | 横流し・なりすまし・リプレイ攻撃 |
| 与信枠の動的制御不能 | 「このエージェントは信頼できるか」を判断できない |

---

## Solution ✅

AgentPass は3つの機能を統合する:

| 機能 | 実現方法 |
|---|---|
| **パスポート** | EdDSA 署名済み JWT — 偽造不可・使い捨て |
| **財布** | CircuitBreaker — 予算・レート制限を cryptographic に証明 |
| **信用スコア** | CreditScorer — 過去の振る舞いに基づく動的スコア |

---

## Business Model

### Wave 1: OSS 配布（現在）

```
無料 OSS → 開発者に先占 → エコシステム構築
収益: なし（市場形成フェーズ）
```

**ゴールドラッシュのスコップ屋戦略:**  
AIエージェント爆増の波が来る前に、インフラを先占する。  
スコップ（インフラ）を持っているプレイヤーが、金（市場）を掘るプレイヤー全員に売れる。

### Wave 2: 信用スコア API 🔭

```
AgentID レピュテーション API
→ 加盟店が「このエージェントは信頼できるか」を API で確認
→ 従量課金 or サブスクリプション

仮説収益モデル:
  - API コール: ¥0.01 / 回
  - プレミアムプラン: ¥50,000 / 月（無制限）
```

### Wave 3: M2M 清算レイヤー 🔭

```
エージェント間微小決済のバッチ清算
→ 清算手数料 0.1%
→ 為替スプレッド
```

### Wave 4: AI 中央銀行 🔭

```
AI 経済圏の基幹インフラ
→ SaaS + プラットフォーム手数料
→ 規制対応コンサルティング
```

## Competitive Analysis 🔭

| 競合 | 強み | AgentPass の優位性 |
|---|---|---|
| Stripe (for AI) | ブランド・既存顧客 | M2Mネイティブ・暗号学的証明 |
| API キー管理サービス | シンプル | リプレイ攻撃防御・使い捨て |
| Web3/ブロックチェーン決済 | 分散・透明性 | 低遅延・シンプル・Python 親和性 |
| 自社実装 | コントロール | 実装コスト削減・セキュリティ保証 |

**差別化要因:**
1. `pip install agentpass` の5分導入
2. EdDSA + JTI による暗号学的証明（ライブラリ不要）
3. 公開鍵自律取得（中央サーバー不要）
4. OSS → 信頼・コミュニティ
